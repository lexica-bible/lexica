# Lexica dictionary — JP quick reference (how to drive the new system)

Written 2026-07-10 (step-4 session), for the low-availability stretch. One page.
The rules behind every line here are in HANDOFF_lexica_rollout.md + AUDIT_lexica_rollout.md —
this page is how to DRIVE, not the law itself.

## The one-list workflow (how everything reaches you now)
- Every session ends with **one decision list**. Each item: the ask in one line, the number it
  changes, evidence below, and a **recommended answer already filled in**. You overrule or say
  "go" — you never have to compose anything.
- **Your silence = pending, never approval.** Nothing fires while an item is open. The engine
  keeps doing side work (tooling, prep, docs) but ships nothing gated.
- Rule when present, any device, days later, in any order. Partial rulings are fine — each item
  stands alone unless the list says two items travel together.
- Flow: CC banks everything in the repo → you paste CC's summary to the reviewer chat → the
  reviewer checks it against the docs → you rule off the list. If the reviewer disputes an
  item, that item goes back to pending; the rest of your rulings stand.

## What a check-in looks like now (two-tier, vs the old way)
- **Old:** you read every card, every flag, every floor.
- **Now:** GREEN words ship with zero read from you when every machine check passes and the
  floor is stable. You see only: (a) AMBER — one flagged region, one glance; (b) RED words —
  the loaded ones, full read, only in your real review windows, they never rush; (c) the
  session's decision list.
- **Defects still reject automatically.** What changed at step 3: harmless wording variety
  that the floor allows ("variance") ships instead of burning redraws — and every such card
  gets a **watch tag** so we can check later whether readers were ever affected. You don't
  review variance ships; the tag is the safety net.
- A render screenshot still goes to the reviewer chat for every ship; you only hear about it
  if it fails.

## When you're actually needed (and when not)
- **Needed (only you):** the decision lists · RED/loaded words · anything touching doctrine,
  money, or a rule change · approving a new batch roster · the final "go" after step-5.
- **Not needed:** floors, draws, gate checks, tooling builds, display work, docs — all run
  without you. If a corpus data problem turns up, the word parks with a DATA-BLOCKED tag and
  work continues elsewhere; the fix reaches you as a normal list item.
- Commands still run on PA by you — CC writes them out verified, you paste. That part is
  unchanged.

## The V8 prompt change (what you'll actually see)
- You will **never be asked to read or paste a prompt.** The V8 draft reaches you as a
  **rendered card to eyeball** — the step-5 control fire draws one word under V8, and you look
  at the card the same way you always have (screenshot to the reviewer chat, 5-point check).
  V7 stays live until you rule on that.
